3. Modèle de données

Schéma Drizzle. Tout doit passer par RLS — un user ne voit que ses propres comptes + les comptes joints où il est membre.

users          (id, email, name, color, initial, created_at)
spaces         (id, user_id, kind: 'perso'|'pro', label)
accounts       (id, space_id, bank, type, name, iban, color,
                initial_balance, currency)
account_members (account_id, user_id, role: 'owner'|'cotitulaire'|'viewer')
                // ← clé pour comptes joints multi-users
categories     (id, space_id, label, kind, color)
incomes        (id, account_id, source, amount, day, type,
                recurrence, created_by)
charges        (id, account_id, label, total, day, split_mode,
                shared, payer_id, created_by)
charge_splits  (charge_id, user_id, amount, settled_at)
                // répartition par coloc, settled quand remboursé
savings_rules  (id, source_account_id, dest_account_id,
                amount, day, label)
purchases      (id, account_id, desc, total, installments,
                monthly, category_id, date)
transactions   (id, account_id, date, amount, label,
                category_id, raw_payload, source: 'manual'|'bank')
shopping_items (id, household_id, label, qty, category,
                priority, est, bought_at, bought_by, added_by)
households     (id, name, address)
household_members (household_id, user_id, share_pct)
invitations    (id, household_id|account_id, email, token,
                accepted_at, expires_at)

5. Comptes joints partagés (priorité)

Le cœur du produit. La table account_members est ce qui rend le compte joint lié à plusieurs utilisateurs réels — pas juste des "noms" dans une coloc fictive.

5.1 — Créer un compte joint Form : nom, banque, type, IBAN optionnel, solde initial. Créateur ajouté en owner.
5.2 — Inviter un co-titulaire Email → row dans invitations avec token. Email envoyé via Resend. Lien d'acceptation → ajoute à account_members en cotitulaire.
5.3 — Synchronisation Realtime Channel Supabase account:{id}. Toute insertion/modif sur charges, transactions, shopping_items broadcast à tous les membres.
5.4 — Répartition automatique À la création d'une charge sur compte joint, générer les charge_splits selon le mode (égal / par parts / par % du household_members.share_pct).
5.5 — Règlement entre coloc Bouton "Marquer remboursé" → set settled_at. Vue récap (déjà mockée dans le proto) calcule "qui doit à qui" en simplifiant le graphe.

    ~1j

9. Setup Claude Code → GitHub

Setup initial pour que Claude Code Web puisse pusher direct sur ton repo. À faire une fois.

9.1 — Créer le repo GitHub

# En local ou via gh.io
gh repo create compte-gestion --private --description "App de gestion budget perso + coloc + pro"
cd compte-gestion
git init && git branch -M main

9.2 — Bootstrap Next.js

npx create-next-app@latest . --typescript --tailwind --app --src-dir=false --import-alias="@/*"
npm i @supabase/supabase-js @supabase/ssr drizzle-orm postgres
npm i recharts framer-motion lucide-react
npm i -D drizzle-kit @types/node vitest @playwright/test
npx shadcn@latest init

9.3 — Variables d'environnement

# .env.local
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=
DATABASE_URL=
GOCARDLESS_SECRET_ID=
GOCARDLESS_SECRET_KEY=
RESEND_API_KEY=
NEXTAUTH_SECRET=

9.4 — Connecter Claude Code Web

  1. Aller sur claude.com/code → "Connect GitHub".
  2. Autoriser l'app Claude sur l'org/le compte qui contient compte-gestion.
  3. Dans le picker de repo, sélectionner compte-gestion + branche main.
  4. Activer "Auto-commit & push" pour que chaque session pousse direct.

9.5 — Premier push depuis ce projet

# Copier ces fichiers du proto comme référence visuelle
git add styles.css data.js pages.jsx shell.jsx app.jsx
git commit -m "design: import prototype reference files"
git remote add origin git@github.com:<you>/compte-gestion.git
git push -u origin main

10. Prompts Claude Code (à copier-coller)

Une fois le repo connecté à Claude Code Web, lancer ces prompts dans l'ordre. Chacun = une PR.

Prompt 1 — Setup Supabase + schema

Crée le schéma Drizzle décrit dans /docs/handoff.html section 3. Génère les migrations Supabase. Active RLS sur toutes les tables. Ajoute les policies décrites en 4.5 — un user ne voit que ses comptes + les comptes joints où il est dans account_members. Tests Vitest qui vérifient l'isolation.

Prompt 2 — Auth & onboarding

Implémente l'auth magic-link Supabase. Onboarding 3 étapes : email → infos profil → création space perso. Reprends exactement la palette + typo du prototype (styles.css à la racine). Mobile-first.

Prompt 3 — Shell + navigation

Porte shell.jsx en composants React Server + Client TypeScript. Sidebar collapsible desktop, bottom nav + drawer "Plus" mobile. Switcher Perso/Pro branché sur cookie + middleware. Tweaks panel avec persistance localStorage.

Prompt 4 — Comptes & comptes joints

Page /accounts depuis pages.jsx (fn AccountsPage). CRUD compte. Modale "Inviter co-titulaire" avec envoi email Resend + page d'acceptation /invitations/[token]. Affichage AvatarStack des membres. RLS testée.

Prompt 5 — Charges, splits, récap coloc

CRUD charges. Trigger Postgres qui crée les charge_splits à l'insert. Page /coloc avec algo de simplification (min cash flow). Realtime channel pour broadcast aux co-titulaires.

Prompt 6 — Dashboard + charts

Porte Dashboard + CalendarPage + YearlyPage. Charts en Recharts. Animations opt-in via Framer (toggle Tweaks). Projections solde 30j basées sur charges récurrentes.

Prompt 7 — GoCardless

Intégration GoCardless Bank Account Data. /api/bank/link flow OAuth. Cron quotidien /api/bank/sync. Catégorisation regex + override. UI de réconciliation charge attendue ↔ transaction réelle.

Prompt 8 — Module pro

Page /compta-pro depuis pages.jsx (fn ComptaProPage). Génération factures react-pdf. Calcul URSSAF auto avec virement vers compte provision. Export FEC. Compteur seuil TVA.

Prompt 9 — Courses temps réel + PWA

Liste courses Realtime. Notifications push web. PWA manifest + service worker. Test installation iOS/Android.
